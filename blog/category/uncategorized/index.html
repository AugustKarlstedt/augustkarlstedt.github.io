How to Fix Inkscape’s Default Units (and Finally Work With Illustrator)

  • Uncategorized
  • Leave a comment

For some odd reason, Inkscape defaults to having 90px = 1in (and soon 96px = 1in). Compared to Illustrator, where 72px = 1in, there are some issues converting back and forth between programs.

There are no configuration settings in Inkscape to change this, but I have found a way to do so anyway.

Navigate to where you’ve installed Inkscape (for me, this is C:\Program Files (x86)\Inkscape), then open the share folder, then open the ui folder. The folder you should be in is C:\Program Files (x86)\Inkscape\share\ui and you should see a file called units.xml.

Make a copy of units.xml called units.bak.xml and then open the original units.xml in a text editor.

Change your file from

All of your units now will calculate correctly! As a test, create a box with size 72px by 72px. Changing the units afterward, you’ll see that this is equal to 1in and correct for the other values as well.

Make sure you also set your default bitmap resolution to 72dpi in Preferences (Shift + Ctrl + P) -> Bitmaps

Another cool feature is that Inkscape supports em units! However, after reviewing the source, the base font size is fixed at 12px. Maybe I can submit a fix for this, hmm.

Happy Inkscaping!

Kali Linux on Raspberry Pi 2

  • Uncategorized
  • Leave a comment

Quick setup instructions.

  1. Download the image located here https://www.offensive-security.com/kali-linux-arm-images/
  2. Extract using 7-zip
  3. Use Rufus to DD the image to your MicroSD
  4. Pop it in the Pi, boot it up, and login using root:toor for the username:password
  5. If you want to install additional tools (you will), you can check them out here https://www.kali.org/news/kali-linux-metapackages/
    1. e.g. apt-get install kali-linux-wireless

Warning: Don’t Install the Windows Subsystem for Linux (Beta) if you have BitLocker Enabled

  • Uncategorized
  • Leave a comment

I found out the hard way.

After upgrading my laptop to Windows 10 Version 1607 (Anniversary Update), I was excited to try Bash in Windows. That’s where my problems started.

After enabling the feature in the “Turn Windows Features On or Off” dialogue, I rebooted my system and was faced with blue screen stating BitLocker encountered a TPM issue.

Resolution: make sure your BIOS is up-to-date and has TPM enabled, then roll back the latest changes using a recovery drive. Finally, manually enter your BitLocker key to boot into Windows, run tpm.msc, and prepare the TPM.
